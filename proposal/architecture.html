⬡ Reports Module — Defect Audit (R1)

Bug
FilterBar ignored — no month/warehouse filtering Reports.vue does not read from useFilters. The global FilterBar selections (period, warehouse, category) have no effect on the Reports page. Backend endpoints accept no query params either.
Gap
No i18n — hardcoded English throughout Reports.vue uses no t() calls. All table headers, labels, and tooltips are hardcoded English strings. Every other module uses useI18n.
Bug
14 console.log() statements in production code The component logs raw API payloads, computed values, and intermediate results on every render cycle, polluting the browser console.
Style
API response format mismatch All other backend endpoints return Pydantic-validated response models. The two Reports endpoints return raw Python dicts, bypassing validation and breaking the established pattern.
Style
Options API instead of Composition API Reports.vue is the only view using data() / methods: / mounted(). All other views use <script setup> with the Composition API.
Style
Duplicate formatNumber() — ignores shared currency.js Reports.vue defines its own number formatter via manual string manipulation. client/src/utils/currency.js already provides formatCurrency() and formatCurrencyWithDecimals().
